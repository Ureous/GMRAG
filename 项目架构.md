# GMRAG 项目架构说明

## 项目概述

**GMRAG** 是一个**多模态检索增强生成（Multimodal Retrieval-Augmented Generation）**项目，旨在通过结合图像和文本的检索来增强多模态问答任务的性能。

## 整体架构

项目采用**两阶段流水线架构**：

```
数据准备 → 检索器训练 → 检索评估 → 多模态问答推理 → 结果评估
```

## 目录结构

```
│

```

## 模块详细说明

### 1. 检索器模块（retriever/）

#### 1.1 数据构造（construct_data/）

**功能**：构建检索器训练所需的正负样本对

**核心文件**：`construct.py`

**工作流程**：
1. **文本分块**：将原始context按行分割成chunks
2. **正负样本标注**：
   - 正样本（pos_chunks）：包含答案的chunks
   - 负样本（neg_chunks）：不包含答案的chunks
3. **相似度筛选**：
   - `simpos()`: 使用**视觉-文本嵌入模型（Visualized_BGE）**，基于查询（query+image）与正样本的相似度，选择最相关的1个正样本
   - `simneg()`: 使用**文本嵌入模型（BGE-M3）**，基于正样本与负样本的相似度，选择最相似的9个困难负样本

**使用的模型**：
- 文本嵌入：`BGEM3FlagModel` (BGE-M3)
- 视觉-文本嵌入：`Visualized_BGE` (基于BGE-base-en-v1.5)

**输出数据格式**：
```json
{
  "voa_example_id": "...",
  "question": "...",
  "positive": ["..."],      # 1个正样本
  "negtive": ["..."],       # 9个负样本
  "caption": "...",
  "voa_image_id": "...",
  "answer": "...",
  "context": "..."
}
```

**训练数据**：
- MultimodalQA: `all_data_posneg.json`
- MuMuQA: `posneg_inans_mumu(bge-m3).json`

---

#### 1.2 检索器训练（train/）

**功能**：训练多模态检索器模型

**核心框架**：FlagEmbedding

**训练位置**：`FlagEmbedding/FlagEmbedding/visual/downstream/`

**训练脚本**：`.bash` 文件包含训练命令，不同数据集使用不同的训练数据

**模型架构**：
- 基于FlagEmbedding的视觉-文本检索框架
- 融合图像和文本的联合嵌入空间

---

#### 1.3 检索评估（eval/）

**功能**：使用训练好的检索器进行检索评估

**核心文件**：`retrieve.py`

**工作流程**：
1. **加载模型**：
   - BGE文本编码器：`BAAI/bge-base-en-v1.5`
   - EVA视觉编码器：`EVA02-CLIP-B-16`
   - 加载训练好的checkpoint：`BGE_EVA_Token.pth`

2. **检索过程**：
   - 查询编码：将问题文本和图像编码为联合查询向量
   - 语料库编码：编码所有文本chunks
   - 相似度计算：计算查询与语料库的相似度
   - Top-K检索：返回最相关的k个文本chunks（默认k=100）

3. **输出结果**：
```json
{
  "data_id": "...",
  "question": "...",
  "caption": "...",
  "candidates": [...],
  "retrieved": ["chunk1", "chunk2", ...]  # Top-K检索结果
}
```

**关键类**：`Flag_bgev_model` - 多模态检索模型封装

---

### 2. 问答推理模块（testqa/）

#### 2.1 模型封装（models/）

**功能**：封装多模态视觉语言模型

**支持的模型**：
- **LLaVA** (`llava.py`): LLaVA-Next模型
  - 模型：`LlavaNextForConditionalGeneration`
  - 处理器：`LlavaNextProcessor`
  
- **Qwen** (`qwen.py`): Qwen2-VL模型
  - 模型：`Qwen2VLForConditionalGeneration`
  - 处理器：`AutoProcessor`

**核心功能**：
- `load_llava()` / `load_Qwen()`: 模型加载
- `llavaInference()` / `Qwen2vlInference()`: 模型推理

---

#### 2.2 MultimodalQA任务（MultimodalQA/）

**功能**：在MultimodalQA数据集上进行问答推理

**核心文件**：
- `mainllava.py`: 使用LLaVA模型推理
- `mainQwen.py`: 使用Qwen模型推理
- `evaluate.py`: 评估脚本

**工作流程**：
1. **加载检索结果**：从检索评估的输出文件中加载Top-K检索到的文本chunks
2. **构建输入**：
   - 图像：从supporting_context获取图像路径
   - 文本：拼接Top-K检索结果（LLaVA用Top-4，Qwen用Top-3）
   - 问题：原始问题文本
3. **模型推理**：
   - 构建包含图像、文本段落和问题的prompt
   - 使用多模态模型生成答案
4. **结果保存**：保存预测结果到JSONL文件

**输入格式**：
- 测试数据：`TextImagelistQ.json` (筛选出的652条数据)
- 检索结果：从retrieve.py生成的检索结果文件

**输出格式**：
```json
{
  "data_id": "...",
  "question": "...",
  "prediction": "...",
  "answer": "..."
}
```

---

#### 2.3 MuMuQA任务（MuMuQA/）

**功能**：在MuMuQA数据集上进行问答推理

**核心文件**：
- `mainllava.py`: 使用LLaVA模型推理
- `mainQwen.py`: 使用Qwen模型推理
- `mmqa_eval.py`: 评估脚本（支持F1和Exact Match指标）

**工作流程**：
1. **加载检索结果**：从`retriever/eval/train_422_mumu_step2900.json`加载检索结果
2. **构建输入**：
   - 图像：MuMuQA数据集图像（路径：`/root/data5/MuMuQA/resize_images263000/`）
   - 文本：Top-7检索结果拼接
   - 问题：原始问题 + Caption
3. **模型推理**：与MultimodalQA类似
4. **结果评估**：
   - 使用`mmqa_eval.py`计算F1和Exact Match指标
   - 支持bridge evaluation模式

**关键区别**：
- MuMuQA使用caption信息
- MuMuQA使用Top-7检索结果（MultimodalQA使用Top-3/4）

## 数据流

### 训练阶段
```
原始数据 (MultimodalQA/MuMuQA)
    ↓
construct_data/construct.py
    ↓
训练数据 (posneg pairs)
    ↓
retriever/train/
    ↓
训练好的检索器模型 (.pth)
```

### 推理阶段
```
测试数据
    ↓
retriever/eval/retrieve.py
    ↓
检索结果 (Top-K chunks)
    ↓
testqa/MultimodalQA/ 或 testqa/MuMuQA/
    ↓
问答结果 (predictions)
    ↓
评估脚本
    ↓
性能指标 (F1, EM)
```

## 技术栈

### 检索相关
- **FlagEmbedding**: 嵌入模型框架
- **BGE-M3**: 文本嵌入模型
- **Visualized_BGE**: 视觉-文本嵌入模型
- **EVA02-CLIP**: 视觉编码器

### 推理相关
- **LLaVA-Next**: 多模态视觉语言模型
- **Qwen2-VL**: 多模态视觉语言模型
- **Transformers**: HuggingFace模型库

### 数据处理
- **PIL/Pillow**: 图像处理
- **JSON/JSONL**: 数据格式
- **NumPy**: 数值计算

## 关键设计特点

1. **两阶段架构**：
   - 阶段1：训练多模态检索器，从大规模语料库中检索相关文本
   - 阶段2：使用检索到的文本+图像+问题，进行多模态问答

2. **困难负样本挖掘**：
   - 使用相似度计算选择困难的负样本，提高检索器训练效果

3. **多模型支持**：
   - 支持LLaVA和Qwen两种多模态模型
   - 便于比较不同模型的性能

4. **多数据集支持**：
   - MultimodalQA：通用的多模态问答数据集
   - MuMuQA：特定领域的多模态问答数据集

## 使用流程

### 1. 训练检索器
```bash
# 1. 构造训练数据
python retriever/construct_data/construct.py \
    --data_path <原始数据> \
    --output_path <输出路径> \
    --emb_model <BGE-M3模型路径> \
    --embv_model <Visualized_BGE模型路径> \
    --image_path <图像目录>

# 2. 训练检索器（在FlagEmbedding框架下）
# 运行 .bash 训练脚本

# 3. 检索评估
python retriever/eval/retrieve.py
```

### 2. 问答推理
```bash
# 使用LLaVA推理
python testqa/MuMuQA/mainllava.py \
    --model_path <LLaVA模型路径> \
    --test_path <测试数据路径>

# 使用Qwen推理
python testqa/MuMuQA/mainQwen.py \
    --model_path <Qwen模型路径> \
    --test_path <测试数据路径>
```

### 3. 结果评估
```bash
python testqa/MuMuQA/mmqa_eval.py \
    --dataset_file <标准答案文件> \
    --prediction_file <预测结果文件>
```

## 总结

GMRAG项目实现了一个完整的**多模态检索增强生成**系统，通过：
1. 训练多模态检索器，从大规模文本语料库中检索相关信息
2. 结合检索到的文本、图像和问题，使用多模态大模型进行问答
3. 在两个主流多模态QA数据集上验证了方法的有效性

该架构具有良好的模块化和可扩展性，便于在不同数据集和模型上进行实验。
