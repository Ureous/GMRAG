## 🔄 核心流程概览

项目采用**两阶段流水线架构**，整体流程如下：

```
┌─────────────────────────────────────────────────────────────────┐
│                     阶段1: 检索器训练与评估                         │
└─────────────────────────────────────────────────────────────────┘
  1. 数据构造 (construct_data/construct.py)
     ↓
  2. 检索器训练 (train/FlagEmbedding/visual/downstream/*.bash)
     ↓
  3. 检索评估 (eval/retrieve.py)

┌─────────────────────────────────────────────────────────────────┐
│                     阶段2: 多模态问答推理                          │
└─────────────────────────────────────────────────────────────────┘
  4. 加载检索结果
     ↓
  5. 多模态模型推理 (testqa/MuMuQA/ 或 testqa/MultimodalQA/)
     ↓
  6. 结果评估 (eval/mmqa_eval.py 或 evaluate.py)
```

## 📁 入口文件清单

### 阶段1: 检索器模块

#### 1. **数据构造入口**
**文件**: `retriever/construct_data/construct.py`
- **功能**: 构建检索器训练所需的正负样本对
- **输入**: 原始数据集JSON文件
- **输出**: 包含正负样本对的JSON文件

#### 2. **检索器训练入口**
**文件**: `retriever/train/FlagEmbedding/FlagEmbedding/visual/downstream/*.bash`
- `node.bash` - MultimodalQA数据集训练脚本
- `nodemmqa.bash` - MuMuQA数据集训练脚本
- **功能**: 训练多模态检索器模型
- **输出**: 训练好的checkpoint文件 (`.pth`)

#### 3. **检索评估入口**
**文件**: `retriever/eval/retrieve.py`
- **功能**: 使用训练好的检索器进行Top-K检索
- **输入**: 
  - 测试数据
  - 训练好的模型checkpoint
- **输出**: 检索结果JSON文件（包含每个问题的Top-K文本chunks）

### 阶段2: 问答推理模块

#### 4. **MuMuQA任务入口**

**LLaVA模型推理**:
- **文件**: `testqa/MuMuQA/mainllava.py`
- **功能**: 使用LLaVA模型进行MuMuQA数据集问答

**Qwen模型推理**:
- **文件**: `testqa/MuMuQA/mainQwen.py`
- **功能**: 使用Qwen2-VL模型进行MuMuQA数据集问答

**评估**:
- **文件**: `testqa/MuMuQA/mmqa_eval.py`
- **功能**: 计算F1和Exact Match指标

#### 5. **MultimodalQA任务入口**

**LLaVA模型推理**:
- **文件**: `testqa/MultimodalQA/mainllava.py`

**Qwen模型推理**:
- **文件**: `testqa/MultimodalQA/mainQwen.py`

**评估**:
- **文件**: `testqa/MultimodalQA/evaluate.py`
- **功能**: 计算list_em和list_f1指标

## 🔍 详细流程分析

### 阶段1: 检索器训练与评估

#### 步骤1: 数据构造 (`construct.py`)

**核心功能**:
```python
# 1. 加载原始数据
filt = read_json(args.data_path)

# 2. 文本分块
context_chunks = context.split("\n")
pos_chunks = [chunk包含答案的chunks]
neg_chunks = [不包含答案的chunks]

# 3. 正样本筛选 (使用多模态相似度)
pos = simpos(query, img_path, pos_chunks, embv_model, num_context=1)
# - 使用Visualized_BGE模型
# - 输入: query + image
# - 输出: 最相关的1个正样本

# 4. 负样本筛选 (使用文本相似度)
neg = simneg(pos, neg_chunks, emb_model, num_context=9)
# - 使用BGE-M3模型
# - 基于与正样本的相似度选择困难负样本
# - 输出: 最相似的9个负样本

# 5. 保存训练数据
append_to_json_file(new_data, args.output_path)
```

**运行命令**:
```bash
python retriever/construct_data/construct.py \
    --data_path <原始数据路径> \
    --output_path <输出路径> \
    --emb_model <BGE-M3模型路径> \
    --embv_model <Visualized_BGE模型路径> \
    --image_path <图像目录>
```

**输出格式**:
```json
{
  "voa_example_id": "...",

  "positive": ["..."],
  "negtive": ["...", "...", ...],  // 9个负样本

  "answer": "...",
  "context": "..."
```

#### 步骤3: 检索评估 (`retrieve.py`)

**核心流程**:
```python
# 1. 加载模型
model = Flag_bgev_model(
    model_name_bge="BAAI/bge-base-en-v1.5",
    model_name_eva="EVA02-CLIP-B-16",
    resume_path="checkpoint-2900/BGE_EVA_Token.pth",
    image_dir="<图像目录>"
)

# 2. 对每个测试样本进行检索
query_embedding = model.encode_queries([query, image], query_type="mm_it")
corpus_embeddings = model.encode_corpus_item(corpus, corpus_type='text')

# 3. 计算相似度并检索Top-K
similarity = query_embedding @ corpus_embeddings.T
top_indices = np.argsort(-similarity)[:k]  # k=100
top_sentences = [corpus[i] for i in top_indices]

# 4. 保存检索结果
```

**输出格式**:
```json
{
  "data_id": "...",
  "question": "...",
  "caption": "...",
  "candidates": [...],
  "retrieved": ["chunk1", "chunk2", ..., "chunk100"]  # Top-K检索结果
}
```

### 阶段2: 多模态问答推理

#### 步骤4: 模型推理 (以MuMuQA为例)

**LLaVA推理流程** (`mainllava.py`):

```python
# 1. 加载模型
model, processor = load_llava(model_path)

# 2. 加载检索结果
ques_data = read_json("retriever/eval/train_422_mumu_step2900.json")
id2candidata = {item["data_id"]: item["retrieved"] for item in ques_data}

# 3. 对每个测试样本推理
for item in test_data:
    # 获取检索到的Top-K文本
    txt_list = id2candidata[item["id"]][:7]  # Top-7
    txt = "\n".join(txt_list)
    
    # 加载图像
    image = Image.open(image_path)
    
    # 构建prompt
    messages = [
        {
            "role": "system",
            "content": [
                {"type": "text", "text": "Caption: " + caption},
                {"type": "text", "text": "Passage: " + txt}
            ]
        },
        {
            "role": "user",
            "content": [
                {"type": "text", "text": "Image: "},
                {"type": "image"},
                {"type": "text", "text": "Question: " + question}
            ]
        }
    ]
    
    # 模型推理
    output = llavaInference(messages, image, model, processor)
    
    # 保存结果
    save_to_jsonl({"data_id": ..., "prediction": output, ...})
```

#### 2. 创建缺失的工具模块

**创建 `my_utils.py`** (放在项目根目录或添加到PYTHONPATH):
```python
import json
import random
import torch
import numpy as np

def set_seed(seed):
    """设置随机种子"""
    random.seed(seed)
    np.random.seed(seed)
    torch.manual_seed(seed)
    if torch.cuda.is_available():
        torch.cuda.manual_seed_all(seed)

def read_json(file_path):
    """读取JSON文件"""
    with open(file_path, 'r', encoding='utf-8') as f:
        return json.load(f)

def read_jsonl(file_path):
    """读取JSONL文件"""
    data = []
    with open(file_path, 'r', encoding='utf-8') as f:
        for line in f:
            data.append(json.loads(line.strip()))
    return data

def append_to_json_file(data, file_path):
    """追加数据到JSONL文件"""
    with open(file_path, 'a', encoding='utf-8') as f:
        f.write(json.dumps(data, ensure_ascii=False) + '\n')
```

## ⚠️ 常见问题

### 1. `my_utils`模块找不到

**解决方案**: 
- 创建`my_utils.py`文件（参考上面的代码）
- 或将其添加到PYTHONPATH

### 2. 模型路径问题

**问题**: 代码中硬编码了绝对路径（如`/root/data1/Code/...`）

**解决方案**: 
- 修改代码中的路径为相对路径或使用环境变量
- 或创建符号链接

### 3. 图像路径问题

**问题**: 图像路径格式不匹配

**解决方案**: 
- 检查图像文件名格式 (如`voa_image_id + '.' + extend`)
- 确保图像目录正确

### 4. GPU内存不足

**解决方案**: 
- 减少`batch_size`
- 使用`gradient_checkpointing`
- 使用更少的GPU或更小的模型

### 5. 检索结果文件格式

**确保检索结果文件格式正确**:
```json
{
  "data_id": "问题ID",
  "retrieved": ["文本chunk1", "文本chunk2", ...]
}
```

### 6. 多模态模型加载失败

**解决方案**:
- 检查模型路径是否正确
- 确保有足够的磁盘空间下载模型
- 检查网络连接（HuggingFace下载）
